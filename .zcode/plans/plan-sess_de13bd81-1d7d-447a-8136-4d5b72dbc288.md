## 问题根因

提交 `551c2b2` 给 iPad 宽屏引入了 `AdaptiveNavigationContainer`,用已废弃的 `NavigationView{}.navigationViewStyle(.columns)` 实现双栏。但会话详情跳转靠 `SessionListView.swift:123` 藏在 `ZStack` 里的隐藏 `NavigationLink(isActive:)`。在 `.columns` 双栏模式下,这种不在 List 内、无 selection 绑定的隐藏 NavigationLink 无法驱动 detail 栏 —— 点会话只更新了 state、详情页不出现。iPhone 走 `.stack` 单栏 push,所以正常。

部署目标 iOS 26,`NavigationView` 已废弃。修复方案:**用 `NavigationSplitView` 正确实现 iPad 双栏**(保留本次提交的设计意图),iPhone 仍为单栏。

## 实现步骤

### 1. `SessionListView.swift` — 改为基于 selection 的导航(核心)

把 `quickOpenSession: SessionSnapshot?` 替换为 `selectedSessionID: String?` 作为选中绑定。

- **列表行点击**(`managedSessionRow`,~L333):由设置 `quickOpenSession = session` 改为设置 `selectedSessionID = session.id`(选择模式下仍是 `toggleSelection`)。
- **隐藏 NavigationLink**(L123-135):整段删除。改为 detail 内容由 SplitView 承载(见步骤 2)。
- **`SessionDestinationView`**(L521):从 `private struct` 提升为可在 NativeRootView 中引用的类型(改可见性即可,仍放本文件)。
- **`quickOpenActive`、`handleQuickAction`、`resume`、`deleteSession`**(L211-244, L400-418, L438-444):把其中对 `quickOpenSession = session` 的赋值改为 `selectedSessionID = session.id`;`deleteSession` 里清空选中的逻辑同步改。
- **新建会话回跳**(`.sheet` 闭合回调,L168-176):`quickOpenSession = newSession` 改为 `selectedSessionID = newSession.id`。
- 新增一个计算属性 `selectedDestination: SessionSnapshot?`,根据 `selectedSessionID` 在 `sessions` 里查找,供 NativeRootView detail 栏使用(或直接暴露 `selectedSessionID`,在 NativeRootView 里查找)。采用后者更简洁,避免两处 state 不同步。

### 2. `NativeRootView.swift` — `AdaptiveNavigationContainer` 改用 `NavigationSplitView` / `NavigationStack`

把 `AdaptiveNavigationContainer` 重写为持有选中状态的形式:

- **宽屏**:`NavigationSplitView(selection: $selectedSessionID)`(`selectedSessionID` 提升到 NativeRootView 的 `@State`),`sidebar` 放当前 `.ready` 阶段的 VStack(SessionListView + toolbar + updateBanner),`detail` 放 `if let id = selectedSessionID, let session = sessions.first(where:{$0.id==id}) { SessionDestinationView(...) } else { placeholder }`。
- **窄屏**:`NavigationStack { ... }`,sidebar 内容(含 SessionListView)作为根,detail 用 `.navigationDestination(isPresented:)` 或 `navigationDestination(item:)` 推入。
- 难点:NativeRootView 当前对 `sessions` 列表无感知(SessionListView 自己持有)。**最干净的做法**:把选中状态 `@State selectedSessionID` 和「当前选中会话快照」都放进 NativeRootView,SessionListView 通过闭包/绑定回传 `sessions` 与点击事件。

**简化决策(避免大改 SessionListView 的数据流)**:让 SessionListView 通过一个新的 `@Binding var selection: String?` 暴露选中态;NativeRootView 持有该 `@State`,并在 detail 栏用一个轻量查找:由于 NativeRootView 拿不到 sessions,改为 detail 栏接收 `selectedSessionID` 后,若需要完整快照,由 SessionListView 在选中时同时把快照回传给 NativeRootView 的另一个 `@State var selectedSnapshot: SessionSnapshot?`。即:点行 → SessionListView 同时写 `selection` 和 `selectedSnapshot`。

### 3. 保留 `usesWideListDetail`(有测试覆盖)

`WandProtocolTests.swift:54-56` 测了 640×480 边界。该函数保留不动,SplitView/Stack 的分支判断仍复用它。

### 4. 处理 toolbar / 覆盖层兼容

- NativeRootView 的 `fullScreenCover`(showWebFallback)、`sheet`(showSettings)、`.task`/`.onAppear` 等修饰符当前挂在 AdaptiveNavigationContainer 上(L115-148),迁移后仍挂在最外层容器上,不变。
- `.navigationBarLeading` toolbar(L83)、SessionListView 的 principal/trailing toolbar(L140/151)、ChatView 的 toolbar —— 在 SplitView 的 sidebar / detail 各自栏位下均有效,`.navigationBarLeading`/`.principal`/`.navigationBarTrailing` 语义不变。
- `detail栏为空时`显示占位视图(图标 + "选择一个会话")。

### 5. 验证

- 跑 `WandProtocolTests` 确认 `usesWideListDetail` 边界用例仍通过。
- 无新增测试(纯 UI 导航修复),改动以 Xcode 编译 + iPad/iPhone 模拟器手测为主。

## 影响文件
- `ios/Wand/SessionListView.swift`(核心:selection 数据流、删隐藏 NavigationLink、提升 SessionDestinationView 可见性)
- `ios/Wand/NativeRootView.swift`(重写 AdaptiveNavigationContainer 为 SplitView/Stack,提升 selectedSnapshot state)

## 风险与权衡
- SplitView 要求把选中状态提升到容器层,会比现在的隐藏 NavigationLink 多一点数据流改动,但这是 `.columns` 双栏 + 程序化跳转唯一可靠的实现方式(NavigationLink(isActive:) 在 columns 下已知不可靠)。
- 不改 sheet 内各自独立的 `.stack` NavigationView(QRScanner/Settings/GitQuickCommit/NewSession),它们与本次问题无关。
