# wand-cli 用户体验测试计划

> 创建时间：2026-03-23
> UX 研究员：Claude (UX Research Agent)

---

## 一、测试目标

1. 验证新用户首次使用流程的流畅性
2. 评估核心功能的易用性和效率
3. 发现界面交互中的痛点和问题
4. 收集用户对新增功能（聊天界面、文件浏览器）的反馈
5. 评估移动端体验

---

## 二、测试用户画像

### 目标用户特征
- **身份**：前端/后端开发者、全栈工程师
- **经验**：1 年以上开发经验，有 CLI 工具使用习惯
- **技术栈**：使用过 Claude Code、Codex 或其他 AI 编程助手
- **设备**：同时使用桌面和移动设备开发

### 招募标准
| 标准 | 要求 |
|------|------|
| 开发经验 | 1-5 年 |
| CLI 使用频率 | 每周至少 3 次 |
| AI 工具经验 | 使用过 Claude Code 或类似工具 |
| 设备 | 有智能手机，愿意测试移动端 |
| 可用性 | 能参与 60-90 分钟测试会议 |

## 三、测试任务清单

### 任务 1：首次使用流程（15 分钟）

**目标**：评估新用户上手的流畅度

**场景**：假设你第一次使用 wand-cli，请完成以下步骤：

1. 安装并初始化 wand-cli
   ```bash
   npm install -g wand-cli
   wand init
   ```

2. 启动 Web 服务
   ```bash
   wand web
   ```

3. 在浏览器中打开 `https://localhost:8443`

4. 使用密码登录

**观察点**：
- [ ] 用户是否能理解配置文件的作用
- [ ] 登录流程是否直观
- [ ] 首次看到主界面时的困惑点
- [ ] 是否需要查看文档才能继续

---

### 任务 2：新建会话（10 分钟）

**目标**：评估新建会话流程的易用性

**场景**：你需要启动一个新的 Claude 会话来编写代码

1. 点击「新对话」按钮
2. 选择「Claude」预设
3. 选择模式（default / full-access）
4. 确认会话启动

**观察点**：
- [ ] 用户是否理解不同模式的区别
- [ ] 模式选择是否直观
- [ ] 是否会困惑于「原生返回」vs「对话返回」

---

### 任务 3：聊天界面使用（20 分钟）

**目标**：评估聊天式对话界面的体验

**场景**：使用聊天界面与 Claude 进行对话

1. 切换到「对话返回」视图
2. 输入一条消息并发送
3. 观察消息渲染效果
4. 尝试复制代码块中的代码
5. 使用 Shift+Enter 换行输入多行

**观察点**：
- [ ] 消息气泡样式是否清晰区分用户/AI
- [ ] Markdown 渲染是否符合预期
- [ ] 代码复制按钮是否容易发现
- [ ] 输入框交互是否符合习惯
- [ ] 是否有消息闪烁或延迟

---

### 任务 4：文件浏览器使用（15 分钟）

**目标**：评估文件浏览器的实用性

**场景**：你需要查看项目中的文件结构

1. 打开侧边栏 Files 标签
2. 展开目录树
3. 浏览不同层级的文件
4. 尝试找到特定文件

**观察点**：
- [ ] 目录展开/折叠是否流畅
- [ ] 文件加载速度是否可接受
- [ ] 是否容易理解当前所在位置
- [ ] 是否觉得缺少搜索功能

---

### 任务 5：会话管理（10 分钟）

**目标**：评估会话管理功能

**场景**：你有多个进行中的会话，需要切换和管理

1. 查看侧边栏会话列表
2. 切换到另一个会话
3. 查看会话详情
4. 尝试删除一个会话

**观察点**：
- [ ] 会话列表是否清晰显示会话状态
- [ ] 切换会话是否流畅
- [ ] 是否理解「Resume」按钮的作用
- [ ] 删除操作是否有确认提示

---

### 任务 6：移动端体验（15 分钟）

**目标**：评估移动设备上的使用体验

**场景**：在手机上远程连接到 wand-cli

1. 在手机浏览器中打开 wand-cli
2. 完成登录
3. 尝试新建会话
4. 使用悬浮窗快捷键
5. 测试横屏/竖屏切换

**观察点**：
- [ ] 页面是否适配小屏幕
- [ ] 触摸操作是否准确
- [ ] 悬浮窗是否容易使用
- [ ] 横竖屏切换是否正常
- [ ] 输入体验是否流畅

---

### 任务 7：快捷键与效率（10 分钟）

**目标**：评估效率功能的使用体验

**场景**：你希望用快捷键提高效率

1. 打开悬浮窗（点击 ⌨ 按钮）
2. 尝试使用 Ctrl+C、Ctrl+D 等快捷键
3. 测试输入框的快捷键（Cmd+C/V/A）

**观察点**：
- [ ] 快捷键提示是否清晰
- [ ] 快捷键响应是否及时
- [ ] 是否容易记住常用快捷键

## 六、问题严重程度分级

| 级别 | 描述 | 示例 |
|------|------|------|
| **P0 - 阻塞** | 导致功能完全无法使用 | 无法登录、无法启动会话 |
| **P1 - 严重** | 显著影响使用体验，但有绕过方法 | 消息闪烁严重、文件加载失败 |
| **P2 - 中等** | 造成困惑或不便 | 按钮含义不清晰、缺少提示 |
| **P3 - 轻微** | 视觉瑕疵或可忽略的问题 | 对齐不精确、颜色不统一 |
